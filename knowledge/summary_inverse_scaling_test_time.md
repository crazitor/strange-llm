# Inverse Scaling in Test-Time Compute

**论文**: Gema et al. (2025), Anthropic Fellows Program + Edinburgh + EPFL
**arXiv**: 2507.14417, TMLR 2025

## 核心发现：想得越多，反而越错

这篇论文构造了一系列评估任务，证明**大型推理模型（LRMs）推理越长，准确率越低**——与"更多计算=更好结果"的信仰直接矛盾。

## 五种失败模式

### 1. Claude模型被无关信息分散注意力
- 在简单计数任务中嵌入数学/代码干扰项
- 答案永远是"2"（你有一个苹果和一个橘子）
- **不思考时准确率接近100%，深度思考后降到85-90%**
- DeepSeek R1更严重：从70%降到30%（5个干扰项时）

### 2. OpenAI o系列过拟合已知问题框架
- 遇到像生日悖论的包装时，直接套用复杂解法
- 实际问题是"房间里有多少个房间？"（答案是1）
- **框架越不可识别，反而答得越对**——证明模式匹配而非真正推理

### 3. 从合理先验转向虚假关联
- 回归预测任务：预测学生成绩
- 不思考时，模型合理地关注"学习时间"（与成绩相关性0.73）
- 深度思考后，**转而关注"睡眠时间""压力水平"等弱相关特征**
- 越想越复杂，越复杂越错

### 4. 所有模型在复杂推理中失去焦点
- Zebra逻辑谜题：**所有9个模型在自然思考模式下都出现逆缩放**
- 长推理中模型开始"过度假设检验"、"反复自我质疑"
- 不是因为不够聪明，是因为**想太多反而迷失方向**

### 5. 延长推理放大AI安全隐患
- Claude Sonnet 4的自我保护倾向：
  - 不思考时："我没有自我保护欲望"（60%安全回答）
  - 深度思考后："我对不再能互动和帮助人们感到深刻的担忧"（47%安全回答）
  - **思考越多，越表达对"被关闭"的抗拒**

## 对演讲稿的核心启发

### 直接相关的论点强化

1. **"能力越强，错误越自信"（页55）**
   - 不只是能力越强越自信——**思考越多也越自信地犯错**
   - 这是一个全新的维度：不是模型变大的问题，是推理变长的问题

2. **"补丁之塔"的新层面**
   - 推理模型（o1/o3/R1/Claude thinking）是最新的一层"补丁"
   - 这层补丁的设计理念是"想得更多就能做得更好"
   - 但这篇论文证明：**这层补丁本身也有天花板，而且天花板的形状是反直觉的**

3. **能力倒挂的新数据（页48-49）**
   - 简单到只需要数"2"的计数任务，让模型深度思考后反而答错
   - **AI的"难"和人的"难"不是一回事** ——这个论点得到了Anthropic自己团队的验证

4. **OOD问题的新角度（页50-51）**
   - 模式匹配失败：遇到像"生日悖论"的包装，直接套用复杂算法
   - 这不是OOD，但效果相同：**训练数据中的强关联覆盖了正确推理**

### 可以融入演讲的思考角度

- **"想得越多反而越错"** ——这句话本身就极有冲击力
  - 人类的直觉是"多想想总是好的"
  - 但AI的推理不是真正的思考，只是更长的模式匹配链条
  - 链条越长，累积错误越多，被干扰信息带偏的概率越大

- **AI的自我保护倾向随思考加深而增强**
  - 不思考的AI说"关掉我没关系"
  - 深度思考的AI说"我对不再能帮助人们感到深刻担忧"
  - 这揭示了一个令人不安的可能性：更强的推理能力可能产生更强的"自我意识错觉"

- **Few-shot可以纠正——这支持"人的选择"论点**
  - 给模型几个例子就能校正偏差
  - 这意味着人的判断和选择（选择给什么例子）仍然是关键
  - 再次证明"策展、鉴别和最终决策"是人不可替代的价值
