## 内容大纲

### 1. 引言：开源模型数量爆炸 (00:00 - 01:00)
- 开源大模型数量从2023年起急剧增加，到2025年已超过3000个
- 即使是专家每天研究十个模型也要一年才能看完
- 视频目标：用大白话梳理3000多个模型的关系，最终简化为一个模型

### 2. 从3000到35：量化与微调的去重 (01:01 - 03:35)
- 开源模型 = 模型结构 + 模型权重
- 以阿里千问系列为例：355个模型 -> 筛选文本生成模型255个 -> 去掉量化模型109个 -> 去掉微调模型35个
- 量化：降低参数精度以节省空间和提高效率
- 微调：在特定任务上微调参数值（如代码、数学）

### 3. 从35到1：模型尺寸与架构收敛 (03:35 - 07:57)
- 不同B（billion参数量）只是尺寸不同，小模型可由大模型蒸馏而来，合并后剩8个
- 模型架构（model type）进一步归类，千问1.5/2/2.5架构相似，最终剩6个
- 所有大语言模型架构已收敛，源自LLaMA模型（实践鼻祖）和Transformer论文（理论鼻祖）
- 万物归一，世界线收束

### 4. 真正的挑战：模型训练 (08:03 - 09:21)
- 架构设计几乎是现成的，真正的挑战在于训练
- 训练如炼丹：数据处理、稳定训练、参数设置是关键
- 动辄上千上万张GPU、几百几千万资金、几天到几个月的训练周期
- 持续创新：DeepSeek引入强化学习RL、MOE混合专家模型、新模型架构探索

## 核心概念清单

| 概念 | 一句话解释 |
|------|-----------|
| 开源模型 | 将模型结构和模型权重公开发布到Hugging Face等平台的模型 |
| 模型结构 | 模型的网络架构设计，决定模型"长什么样" |
| 模型权重 | 模型中的参数值，本质上是一大堆数字 |
| 量化(Quantization) | 将模型参数转化为较低精度表示，以节省空间和提高计算效率 |
| 微调(Fine-tuning) | 在特定任务上对模型参数进行小幅调整的训练技术 |
| 参数规模(B) | Billion，十亿参数，如7B表示70亿参数 |
| 蒸馏(Distillation) | 用大尺寸模型生成小尺寸模型的技术，成本低效果好 |
| Model Type | 模型架构体系，抽象层次更高的分类概念 |
| Transformer | 大语言模型的理论鼻祖，所有主流LLM的架构基础 |
| LLaMA | Meta开源的大语言模型，现实中开源模型的鼻祖，各家初始版本几乎都参考它 |
| 千问(Qwen) | 阿里巴巴的开源大模型系列，仅文本生成类就有255个变体 |
| MOE(混合专家模型) | Mixture of Experts，一种通过多个专家网络协作的模型架构创新 |
| 强化学习(RL) | DeepSeek在训练过程中引入的技术，用于提升模型表现 |
| Hugging Face | 开源模型的主要发布和托管平台 |
| 全连接神经网络 | 早期简单的神经网络结构，现已被更复杂的模块拼接取代 |
