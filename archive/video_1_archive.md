## 内容大纲

### 1. 大语言模型基础概念（00:00 - 02:26）
从语言模型的发展讲起，介绍LLM（大语言模型）的本质是"文字接龙"，通过引入一问一答的对话模式、Prompt、Context（上下文）、Memory（记忆）等概念，将一个只能做词语接龙的模型变成可以持续对话的"员工"。

### 2. Agent与工具调用（02:26 - 05:05）
当LLM无法独立完成联网搜索等任务时，引入中间代理程序（Agent）来代理沟通并完成搜索任务。进而介绍RAG（检索增强生成）通过向量数据库进行语义匹配，以及Function Calling作为Agent与大模型之间关于工具调用的格式约定。

### 3. MCP协议与交互形式（05:05 - 06:53）
将工具实现从Agent主程序中解耦为独立服务，引入MCP（模型上下文协议）作为Agent与工具服务之间的调用规范。同时介绍了Agent可以有多种交互形式：命令行、IDE、桌面助手（如Claude Code / Claude Bot）。

### 4. Workflow、Skill与流程固化（06:53 - 09:18）
面对复杂任务，讨论如何将稳定流程固化：通过编程框架LangChain、低代码拖拽Workflow、以及Skill（提示词+脚本的目录包）三种方式。还介绍了Sub Agent作为上下文隔离手段。

### 5. 概念关系梳理与未来展望（09:18 - 14:39）
高屋建瓴地梳理所有概念之间的关系：Function Calling与MCP不搭界、Skill与MCP不在同一维度。提出"从刚性到柔性"的光谱（LangChain -> Workflow -> Skill -> 纯Agent）。展望未来token将越来越便宜，一个打包好的"超级Agent"将让普通人无需配置即可使用。

## 核心概念清单

| 概念 | 一句话解释 |
|------|-----------|
| LLM（大语言模型） | 参数规模足够大后涌现出智能的语言模型，本质是不断预测下一个字（文字接龙） |
| Prompt（提示词） | 每次和大模型对话时输入的全部内容 |
| Context（上下文） | Prompt中属于背景信息的部分，用于给大模型提供参考 |
| Memory（记忆） | 将历史对话放入Context中伪装成多轮对话，实现追问能力 |
| Agent（智能体） | 介于用户和大模型之间的程序，代理沟通并操作工具，本质是"所有不需要智能的部分" |
| RAG（检索增强生成） | 通过向量数据库进行语义匹配，将检索到的信息加入上下文以增强生成质量 |
| Web Search（联网搜索） | 让Agent具备上网搜索实时信息的能力 |
| Function Calling | Agent与大模型之间关于工具调用所约定的对话格式（如JSON），方便程序解析 |
| MCP（模型上下文协议） | Agent与外部工具服务之间的调用规范，约定如何发现工具、调用工具、传参和接收返回值 |
| LangChain | 纯编程的链式任务框架，稳定但缺乏柔性 |
| Workflow（工作流） | 低代码拖拽方式来编排多步任务流程 |
| Skill（技能） | 提前写好的说明文档（skill.md）和可运行脚本的目录包，本质是提示词加载器 |
| Sub Agent（子智能体） | 用于处理独立子任务的隔离Agent，其产生的上下文不会保留在主Agent中 |
